## Polkadot testnets
* **Paseo (official)** — Web faucet; drips **PAS** to any valid SS58 address.
  [https://faucet.polkadot.io/](https://faucet.polkadot.io/)

* **Westend** — Web faucet; drips **WND**; good for low-level Substrate testing.
  [https://faucet.polkadot.io/westend](https://faucet.polkadot.io/westend) · Alt: [https://faucet.triangleplatform.com/polkadot/westend](https://faucet.triangleplatform.com/polkadot/westend)

* **Rococo** (legacy parachain testnet) — Often used by parachains and contracts testnets.
  [https://faucet.triangleplatform.com/polkadot/rococo](https://faucet.triangleplatform.com/polkadot/rococo)

* **ink! Contracts (Rococo parachain)** — Uses Rococo tokens (**ROC**) for contract deploy/tests. Check ink! docs for the current faucet endpoint.

* **Frequency (testnet)** — Dedicated faucet UI for **tFRQCY**.
  [https://faucet.testnet.frequency.xyz/](https://faucet.testnet.frequency.xyz/)

## Usage notes & troubleshooting
* **Address format:** Use the correct **SS58** or **EVM** address for the network you’re targeting.
* **Cooldowns & caps:** Many faucets enforce **per-address** and **per-IP** limits. Wait the cooldown or try again later.
* **Bots & verification:** Some Discord/Telegram faucets require Captcha or account age checks.
* **Bridging test funds:** Avoid trying to bridge testnet tokens between ecosystems; use each network’s faucet instead.
* **Stuck claims:** If a claim “succeeds” but balance doesn’t update, confirm you’re viewing the **right network/testnet** in your wallet, then check the explorer for your address.
